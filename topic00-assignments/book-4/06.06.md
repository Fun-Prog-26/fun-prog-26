# 6. Marking Scheme  (indicative)

This assignment is worth 40% of you module. 

The marks (100) are broken down into the following :

| Component       | Marks |
|-----------------|-------|
| Appropriateness of data design (entities, datatypes, csv files)       | 13   |
| Validation of entity fields (including primary keys)                                      | 18 |
| Validation of foreign fields                                       | 4  |
| Validation of timetable entry as per validation tab                |   16  |
| Validation of lecturer's hours limit                                  | 3| 
| Generation of documents               |   27  |
| Programming style                                                  | 13  |
| Documentation (including reflection)                               | 6   |
| Extras  (Only applicable if the main functionality  has been implemented )                                                          | 10    | 
|-------------------------------------|------|
| Total           | 100   |

You can also get a much more detailed version as a soft copy from here: 
[Marking Scheme](archives/markingScheme.xlsx)

## Extras
1. You can get up to 10 extra marks for extra functionality that you clearly document in readme.md
2. You can only receive marks for these extras iff you have attempted all the functionality (mandatory and optional) in the spec. 
3. The total of your extra marks and other marks will be capped at 100%

Some changes may be made to this marking scheme but any such changes are unlikey to be significant.
